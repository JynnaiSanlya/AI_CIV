# 版本更新公告 - 0.1.2 (2026-03-14)

## 主要改动

### 1. AI多决策支持
- **核心功能**：允许AI在一个回合内同时执行多个决策
- **决策数量**：AI可以根据需要执行任意数量的动作
- **执行顺序**：动作按照AI提供的顺序执行
- **资源限制**：所有动作的资源消耗总和不能超过现有资源

### 2. 提示词优化
- 更新了AI提示词，允许选择多个动作
- 示例展示了多个动作的格式
- 明确了动作执行顺序的重要性

### 3. 决策处理改进
- 修改了`get_decision`方法，返回动作列表而非单个动作
- 更新了`handle_civilization_turn`方法，支持处理多个动作
- 每个动作都有独立的执行结果和历史记录

### 4. 游戏效果
- AI可以更灵活地规划发展策略
- 可以同时发展人口、军队和科技
- 更真实地模拟了文明领导者的决策过程
- 增加了游戏的策略性和趣味性

## 技术实现

### 核心修改

1. **ai_controller.py**：
   - 更新了提示词，允许多个动作
   - 修改了决策提取逻辑，支持多个动作
   - 返回动作列表而非单个动作

2. **game.py**：
   - 更新了回合处理逻辑，支持多个动作
   - 每个动作都有独立的输出和历史记录
   - 保留了资源限制机制

### 示例输出

```
Atlantis's Turn (aliyun AI):
Action 1: gather_resources 1
Result: Gathered resources +10
Action 2: grow_population 2
Result: Grew population +2 (cost: 20)
Action 3: develop_technology 1
Result: Developed technology +1 (cost: 20)
```

## 游戏机制

### 动作执行顺序
- AI提供的动作按照顺序执行
- 前一个动作的结果会影响后一个动作
- 资源不足时，后续动作会失败

### 资源管理
- 所有动作共享当前回合的资源
- 动作失败后，后续动作仍会尝试执行
- 资源不足的动作会显示"Action failed - insufficient resources"

### 历史记录
- 每个动作都有独立的历史记录
- 记录了动作类型、数量、结果和执行后的状态

## 配置说明

- 无需额外配置，默认支持AI多决策
- 可在`game.py`中修改`max_turns`调整游戏回合数
- 可在`ai_controller.py`中修改提示词和API参数
