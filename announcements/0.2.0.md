# 版本更新公告 - 0.2.0 (2026-03-14)

## 重大更新

### 1. Web图形界面
- **实现了完整的Web图形界面**，无需安装额外依赖
- **游戏运行在本地HTTP服务器上**，可通过浏览器访问
- **访问地址**：http://localhost:8000
- **文明可视化**：根据不同时代显示不同的文明形象
- **资源图标**：为每种资源添加了独特的颜色和图标
- **外交系统可视化**：显示文明间的外交关系和行动
- **响应式设计**：自适应不同屏幕尺寸，实时更新游戏状态

### 2. 游戏机制完善
- **忠诚度恢复机制**：资源充足时，忠诚度微小恢复（1点/回合）
- **战狂限制机制**：每个时代（除最后一个时代）每方最多主动发起2次战争
- **贸易鼓励机制**：
  - 科技低的一方：根据科技差获得微小科技提升（最高1点/贸易）
  - 科技高的一方：获得1点忠诚度提升
- **资源收集动态上限**：基于人口和科技的动态上限，防止过度资源增长

### 3. AI决策优化
- **行动点系统**：AI决策受行动点限制，每回合5点基础行动点，每进入新时代+1点
- **多决策支持**：AI每回合可以做出多个决策，根据行动点合理分配
- **资源收集上限**：AI了解资源收集的动态上限，做出更合理的决策

## 技术架构

### 1. 代码优化
- **创建了`web_gui.py`**：实现了Web服务器和图形界面
- **游戏逻辑与图形界面分离**：支持多人同时查看游戏状态
- **HTTP服务器和JSON API**：实现游戏状态通信
- **前端技术栈**：HTML、CSS和JavaScript实现动态界面

### 2. 架构调整
- **模块化设计**：游戏逻辑、AI决策和图形界面分离
- **可扩展性**：支持未来添加更多功能和游戏机制
- **多设备支持**：可以通过浏览器在多设备上查看游戏

## 游戏机制概览

### 核心系统
- **资源管理**：资源生产、消耗和分配
- **人口动态**：增长、消耗和忠诚度
- **军事系统**：军队建设和战争机制
- **科技发展**：科技研发和文明进步
- **外交系统**：贸易、战争和外交关系
- **行动点系统**：基于时代的行动点分配

### 文明发展
- **5个时代**：Primitive → Ancient → Medieval → Modern → Future
- **时代奖励**：每个时代提供不同的资源生产和行动点奖励
- **文明形象**：根据时代显示不同的文明形象

### 运行方式
```bash
python web_gui.py
```

然后在浏览器中访问：http://localhost:8000
