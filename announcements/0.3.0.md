# 版本 0.3.0 更新日志

发布日期：2026-03-18

## 1. 内部事件系统

### 1.1 事件机制
- **4种新的文明内部事件**：
  - 民间好战之风：民间出现好战风气，AI需选择鼓励或打压
  - 本国新发明：科学家发明新技术，AI需选择支持或商业化
  - 本国文化运动：文化运动兴起，AI需选择支持新文化或维护传统
  - 发现大量资源：发现资源，AI需选择放任民间开发或收归国有

### 1.2 事件效果
- **多种效果类型**：
  - 即时效果：立即生效的效果（如获得资源、科技等）
  - 持续效果：持续多个回合的效果（如资源增长加速、忠诚度变化等）
- **效果时长限制**：最长不超过5回合
- **平衡的效果设计**：每个决策都有不同的风险和收益

### 1.3 事件触发机制
- **内置冷却时间**：防止事件过于频繁发生
- **基于概率的触发**：每个事件都有触发概率
- **每文明独立事件**：事件是每个文明独立的，不影响其他文明

### 1.4 事件历史记录
- **每个文明独立的事件历史**：记录该文明发生过的所有内部事件
- **Web UI展示**：在Web界面中显示每个文明的事件历史

## 2. 多API提供商支持

### 2.1 MiniMax API集成
- **新增MiniMax API支持**：除了阿里云百炼模型外，现在还支持MiniMax API
- **灵活的模型配置**：可以为每个AI指定不同的API提供商
- **自动API密钥管理**：从配置文件中读取相应的API密钥

### 2.2 配置文件更新
- **新增minimax_api_key字段**：在config.json中添加了MiniMax API密钥配置
- **向后兼容**：现有配置文件无需修改即可继续使用

### 2.3 API调用优化
- **添加超时机制**：API调用设置了10秒超时，防止程序卡死
- **完善的错误处理**：处理API调用失败的情况，使用默认行为
- **标准化的API响应处理**：统一处理不同API提供商的响应格式

## 3. Web界面改进

### 3.1 可折叠UI部分
- **实现可折叠的UI区域**：文明状况、外交、时代事件等部分可以折叠
- **优化显示空间**：减少信息过载，让用户可以专注于感兴趣的内容
- **更好的移动设备支持**：在小屏幕设备上也能良好显示

### 3.2 内部事件UI
- **每个文明独立的内部事件历史**：在Web界面中显示每个文明的内部事件
- **事件详情展示**：显示事件名称、描述、决策和效果
- **美观的事件卡片**：使用卡片式设计展示事件

### 3.3 响应式设计优化
- **改进了响应式布局**：适配不同屏幕尺寸
- **优化了资源显示**：使用更直观的进度条显示资源
