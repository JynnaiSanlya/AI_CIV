# AI Civilization Simulation Game

## 游戏概述
这是一个人工智能模拟人类文明发展的游戏，让两个AI模型控制的文明从原始时期发展到未来，相互竞争和影响。

## 功能特点
- 两个AI文明相互竞争发展
- 支持多API提供商：阿里云百炼（qwen-flash、qwen-plus）和MiniMax API
- 模拟文明的多个维度：资源、人口、军队、科技、忠诚度、文化
- **内部事件系统**：随机发生的文明内部事件，AI需做出决策
  - 4种不同类型的内部事件
  - 支持即时和持续多回合效果
  - 每文明独立的事件历史记录
- **Web图形界面**：实时可视化文明发展（http://localhost:8000）
  - 显示文明每一回合采取的行动
  - 显示每个文明使用的AI模型
  - 外交历史记录与图标
  - 文明旗帜区分
  - 资源图标改进
  - 不同时代的文明图像
  - 可折叠的UI部分，优化显示空间
  - 每个文明独立的内部事件历史
- **行动点系统**：每回合5点基础行动点，每进入新时代+1点
- **文明间交互**：贸易、战争、文化影响
- **外交系统**：支持文明间的贸易和战争
- **自动保存游戏状态**
- **详细的游戏日志和最终总结**
- **图片资源**：时代背景图片使用智谱清言生成，增强视觉体验

## 配置要求
1. Python 3.8+
2. 安装依赖：`pip install -r requirements.txt`
3. 配置API密钥：在`config.json`文件中填写您的API密钥

## 如何运行

### 控制台模式
```bash
python game.py
```

### Web图形界面模式
```bash
python web_gui.py
```
然后在浏览器中访问：http://localhost:8000

## 游戏机制

### 文明属性
- **资源**：用于发展科技、建造军队、增加人口和发展文化
- **人口**：影响资源生产、文化增长和文明规模
- **军队**：保护文明、进行战争和资源劫掠
- **科技**：决定文明的发展阶段和能力
- **文化**：影响忠诚度、吸收人口和进行文化劫掠
- **忠诚度**：影响文明的稳定性和发展
- **行动点**：限制每回合的行动数量
- **时代**：根据科技水平自动升级（原始→古代→中世纪→现代→未来）

### 行动点系统
- 基础行动点：5点/回合
- 每进入新时代+1点行动点
- 不同行动消耗不同的行动点：
  - 发展科技：2点
  - 建造军队：2点
  - 增长人口：1点
  - 收集资源：1点
  - 发展文化：1点

### AI决策
每个文明由不同的AI模型控制，AI会根据当前文明状态、对手情况和可用行动点做出决策。

### 可用行动
1. **发展科技**：消耗资源提升科技水平（2点行动点）
2. **建造军队**：消耗资源增加军事力量（2点行动点）
3. **增长人口**：消耗资源增加人口数量（1点行动点）
4. **收集资源**：增加资源储备（1点行动点，动态上限）
5. **发展文化**：消耗资源提升文化水平（1点行动点）

### 文明间交互

#### 贸易系统
- 自动触发贸易提议
- 科技低的一方：获得微小科技提升（最高1点/贸易）
- 科技高的一方：获得1点忠诚度提升

#### 战争系统
- 自动触发战争
- 战狂限制：每个时代每方最多主动发起2次战争
- 战争对忠诚度的影响：
  - 进攻并获胜：忠诚度+5
  - 进攻并失败：忠诚度-10
  - 防御并获胜：忠诚度+5
  - 防御并失败：忠诚度-10
- 获胜方可以掠夺：
  - 最多40%的资源和人口
  - 最多20%的文化

#### 文化影响
- 文化高的文明会影响文化低的文明
- 影响效果：
  - 文化高的文明：忠诚度提升
  - 文化低的文明：忠诚度下降
  - 文化高的文明：吸收少量人口（最多2人/回合）

### 内部事件系统

#### 事件类型
1. **民间好战之风**：民间出现好战风气，AI需选择鼓励或打压
2. **本国新发明**：科学家发明新技术，AI需选择支持或商业化
3. **本国文化运动**：文化运动兴起，AI需选择支持新文化或维护传统
4. **发现大量资源**：发现资源，AI需选择放任民间开发或收归国有

#### 事件效果
- **即时效果**：立即生效的效果（如获得资源、科技等）
- **持续效果**：持续多个回合的效果（如资源增长加速、忠诚度变化等）
- **效果时长**：最长不超过5回合

#### 事件决策
- AI需根据当前文明状态和事件详情做出决策
- 决策会影响文明的发展轨迹
- 每个决策都有不同的风险和收益

### 资源收集动态上限
- 基于人口和科技的动态上限
- 公式：`max(1, population // 10 + technology * 2)`
- 防止过度资源增长

## 游戏结束条件
- 达到最大回合数（默认10回合）
- 一个文明的科技水平达到100且是对手的两倍以上
- 一个文明的人口崩溃（人口≤0）
- 一个文明的忠诚度降至0

## 配置文件
在`config.json`中配置API密钥：
```json
{
    "anthropic_api_key": "your_anthropic_api_key_here",
    "openai_api_key": "your_openai_api_key_here",
    "aliyun_api_key": "your_aliyun_api_key_here",
    "aliyun_api_secret": "your_aliyun_api_secret_here",
    "minimax_api_key": "your_minimax_api_key_here"
}
```

## 使用阿里云百炼模型

### 配置阿里云API密钥
1. 登录阿里云百炼控制台，获取API密钥
2. 在`config.json`中填写`aliyun_api_key`和`aliyun_api_secret`

### 使用qwen-flash和qwen-plus
游戏默认使用阿里云的qwen-flash和qwen-plus模型。如果需要修改模型，可以在`game.py`中修改初始化参数：

```python
# 默认配置（阿里云模型）
game = CivilizationGame(
    model_name1="qwen-flash",
    model_name2="qwen-plus"
)
```

## 使用MiniMax API

### 配置MiniMax API密钥
1. 登录MiniMax控制台，获取API密钥
2. 在`config.json`中填写`minimax_api_key`

### 使用MiniMax模型
要使用MiniMax API，需要在初始化`CivilizationGame`时指定`model_type`参数：

```python
# 混合配置（一个阿里云模型，一个MiniMax模型）
game = CivilizationGame(
    model_name1="qwen-flash",
    model_type1="aliyun",
    model_name2="abab6.5s-chat",
    model_type2="minimax"
)
```

- 自适应不同屏幕尺寸
